HOW THE EMULATOR WORKS:
  firebase.json declares the RTDB emulator on port 9000.
  VITE_USE_EMULATOR=true tells firebase.js to call connectDatabaseEmulator.
  playwright.config.js passes that env var to the Vite child process.
  window._atpTest.adminWrite/adminRead replace [ADMIN] Firebase console steps in tests.
  Data resets between test suites via clearData() / beforeAll / afterAll hooks.
  To run the emulator standalone: npm run emulator

DEV vs PROD — HOW TO STAY SAFE
═══════════════════════════════════════════════════════════════════════

RULE: `npm run dev` → DEV mode.  `npm run build` → PROD build.

DEV mode prefixes all Firebase paths with `_dev/`. Prod writes to root.
They NEVER mix.

HOW TO KNOW WHICH MODE YOU ARE IN:
  - A yellow "DEV" badge appears in the top-left of the app bar.
  - No badge = production build.

SETUP FOR MANUAL FLOWS (run once before testing):
  1. npm run dev
  2. Open http://localhost:5173/atp-greenwich/
  3. [CONSOLE] window._atpTest.seedTestData()    — for Flows 1 + 2
  4. [CONSOLE] window._atpTest.seedLeague()      — for Flows 7 + 8 + 9
  5. [CONSOLE] window._atpTest.app()             — jump into app shell

TO RESET DEV DATA:
  [CONSOLE] window._atpTest.clearLeague()  then re-run seedLeague()

KNOWN BEHAVIORS (not bugs)
═══════════════════════════════════════════════════════════════════════
- Invite code error for USED-9999 only appears after tapping "Verify Code" — correct.
- Avatar "Checking…" only fires on "This is me →", not on every shuffle — correct.
- Chunk size warning during build is expected — DiceBear SVG libraries are large.
- Feed and Standings listeners are cleaned up on tab switch (no memory leak).
- Standings ELO rankings only show players with status="active" and a set eloRating.
- clearLeague() preserves invite codes (_dev/invite_codes/); only league/player data cleared.
- Admin app and Companion app both share the same admin password (atpgreenwich2026 default).
- Companion app uses adminOverride:true on submitted results — matches bypass the photo step.
- Bracket tab (player app) live-updates during qualification phase via dbListen; static render once bracket is generated.
- Admin app is never deployed; it runs locally only (npm run dev:admin → localhost:5174).
- Companion app can be deployed as a PWA but there is no deploy script for it yet.
